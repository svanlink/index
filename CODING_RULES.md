# CODING_RULES.md

- Use TypeScript for frontend code.
- Use PascalCase for React components.
- Use camelCase for functions and variables.
- Keep domain logic out of React presentation components.
- Prefer reusable hooks or store modules for shared state logic.
- Keep Tauri/Rust scan code isolated from UI code.
- Do not add authentication.
- Do not add team features.
- Do not add filesystem mutation behavior.
- Do not make assumptions beyond the spec.
- Preserve user data rather than deleting it when uncertain.
- Keep UI minimal, polished, and professional.
